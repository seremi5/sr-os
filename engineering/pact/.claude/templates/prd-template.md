## Formatting Rules

- Single `#` for main sections
- `###` for subsections
- `**bold**` for functional area names within P0
- `---` divider after each main section header
- Plain bullet points — no checkboxes
- `> [Note]` for callout blocks
- `🔴 Open` / `🟡 Medium` / `🟢 Resolved` for status indicators
- `**Integration with [team] required**` for cross-team dependencies
- `@[Name]` for tagging owners
- `++` in last row of Risks table to indicate list is not exhaustive
